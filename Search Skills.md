## Task 4 Specialized Search Engines
### Shodan
### Specialized search engines are designed to find specific types of information rather than general web results
### Shodan is a search engine for Internet-connected devices
### It helps discover servers, networking equipment, industrial systems, and IoT devices
### For example, you can search for “apache 2.4.1” to find servers running that version across the Internet
<img width="1920" height="868" alt="image" src="https://github.com/user-attachments/assets/47671d17-6caf-4312-b6e5-b53a60fc90c6" />

### Censys
### Censys is similar to Shodan but focuses more on Internet-connected hosts, websites, certificates, and other Internet assets
### Shodan focuses on devices like servers, routers, webcams, and IoT devices, while Censys focuses on broader Internet infrastructure
### Censys can be used to enumerate domains, audit open ports and services, and discover rogue assets in a network
### You can also explore Censys introductory use cases to understand its capabilities
<img width="1685" height="932" alt="image" src="https://github.com/user-attachments/assets/aae4b409-e9cf-439d-a5f6-ad9d8c8cdeea" />

### VirusTotal
### VirusTotal is an online service that scans files and URLs using multiple antivirus engines
### It allows users to upload files, submit URLs, or check file hashes for previous scan results
### It compares results across many antivirus engines in a single scan to detect possible malware
### Community comments help provide additional context when a file is falsely flagged as malicious

### Task 5 Vulnerabilities and Exploits
### CVE (Common Vulnerabilities and Exposures) 
### It is a standardized system for identifying security vulnerabilities in software and hardware
### It assigns each vulnerability a unique ID, such as CVE-2024-29988, so everyone refers to the same issue
<img width="1920" height="1080" alt="image" src="https://github.com/user-attachments/assets/5eb9b21d-b830-47ba-8209-7948a48f58e5" />

### CVE is maintained by The MITRE Corporation
### It acts like a dictionary of known vulnerabilities used by researchers, vendors, and security professionals
### You can search CVEs using databases like the CVE Program and the National Vulnerability Database (NVD)
### A well-known example is CVE-2014-0160 (Heartbleed vulnerability)

### Exploiting vulnerable applications
### It is done for security testing, such as red team assessments, but only with proper permission
### Unauthorized exploitation is illegal and should never be attempted
### Exploit Database is a resource that provides exploit code for known vulnerabilities
### It contains exploits written by different authors, some of which are tested and marked as verified
### It is used by security professionals to understand and validate vulnerabilities
<img width="1920" height="1080" alt="image" src="https://github.com/user-attachments/assets/2171a2b2-b3be-41d9-97f5-2fe9536e8982" />

### GitHub 
### It is a web-based platform for software development that also contains security tools and exploit research
### It includes repositories with CVE-related tools, proof-of-concept (PoC) code, and exploit code
### Security researchers often use GitHub to study vulnerabilities like Heartbleed and understand how they work
### Searching GitHub can reveal many public examples and demonstrations of known security issues

## Task 6 Technical Documentation
### Linux Manual Pages
### One important skill in cybersecurity is knowing how to read official documentation
### Linux and Unix-like systems provide built-in documentation called manual pages (man pages)
### Man pages existed before the Internet and are used to get help for commands
### They also document system calls, library functions, and configuration files
### To view a man page, you can use the command: man ip
